# Development Governor v0.1.0-beta.3

Status: **experimental; not production-ready**.

beta.3 closes the three highest-priority enforcement gaps found in beta.2 and separates
semantic Skills from the deterministic runtime authority plane. It is a bounded
hardening release, not a claim that the Governor can judge whether a specification is
correct or a project is worth pursuing.

## What changed

- Evidence inputs are `{path, sha256}` records and are rechecked before start,
  isolated check, and verification.
- Opaque mutation-capable shell commands fail closed under an active lease. Tests and
  other unknown-write-set commands use a non-promoting disposable-snapshot check.
- Frozen acceptance runs in disposable snapshots and cannot ordinarily rewrite the
  governed repository through relative paths.
- Policy migration and default-runtime upgrade require an exact current hash, no active
  lease, an external Owner reference, integrity checks, and immutable receipts.
- Skill promotion accepts independently verified project receipts. New installation
  additionally requires `--allow-new-install`; replacement remains the default.
- `authoring-high-stakes-specs`, `controlling-design-drift`, and
  `reviewing-specs-before-acceptance` now have non-overlapping draft, analysis, and
  review authority. None can create a lease or Owner decision.

## Frozen local evidence

- `BETA3_CORE_ACCEPTANCE=PASS`
- `BETA3_OPS_ACCEPTANCE=PASS`
- `BETA3_SKILL_PROMOTION_ACCEPTANCE=PASS`
- `BETA3_SKILL_GOVERNOR_INTEGRATION=PASS`
- Unit regression: `143/143` passed in isolated verification.

The integration artifact is explicitly synthetic. It content-binds one fixture spec,
checks the installed Skill boundaries, then prepares a distinct implementation capsule
inside disposable acceptance state without starting a lease or executing product
mutation. It does not simulate a real Owner decision.

## Remaining boundaries

- The Hook is not an OS sandbox and cannot stop unhooked tools or malicious absolute
  writes outside the governed repository.
- Owner references are preserved for audit but are not cryptographically authenticated.
- A green receipt proves the frozen checks passed; it does not prove semantic
  correctness, product value, or broad compatibility.
- No measured productivity or cost-saving claim is made by this release.
