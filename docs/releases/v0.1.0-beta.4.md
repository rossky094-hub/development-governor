# Development Governor v0.1.0-beta.4

Status: **experimental prerelease candidate; not production-ready**.

beta.4 closes one deployment-parity gap exposed by real multi-project use and carries
the previously unshipped capsule role-disjointness repair. It remains a bounded control
layer, not a claim that model-generated code or specifications are correct.

## What changed

- Task preparation rejects exact and ancestor/descendant overlap between mutable
  deliverables and immutable evidence inputs.
- A source-bound activation projects `current`, `upgrade_required`, or
  `source_unavailable` from the bound checkout and active runtime hashes.
- The Hook denies external-project mutation for the latter two states before consulting
  the project lease.
- The Governor repository and manifest-bound upgrade command remain available, avoiding
  a self-repair deadlock.
- Source-unbound installed packages keep their existing local integrity model and never
  perform remote release discovery.

## Verification state

The release candidate is awaiting the frozen Governor acceptance run. Publication is
not authorized by this document alone.

## Remaining boundaries

- The Hook is a guardrail for supported Codex mutation paths, not an OS sandbox.
- Source parity detects a local deployment mismatch; it does not prove that the source
  is correct, approved by cryptographic identity, or newer than a remote release.
- Owner references are preserved for audit and are not authenticated.
- A green receipt proves only the frozen checks that ran. It does not prove semantic
  correctness, product value, cost savings, or broad compatibility.
