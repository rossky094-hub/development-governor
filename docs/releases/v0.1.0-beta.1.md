# Development Governor v0.1.0-beta.1

This is the first public beta of a deterministic control layer for bounded Codex
development work.

## Why it exists

The project began after repeated observations of long-running agent tasks that spent
their budget on review loops, interrupted reruns, documentation churn, or tests whose
success boundary could drift with the implementation. The beta turns a small subset
of that diagnosis into executable controls rather than another review workflow.

## What is included

- Installable `governor` and `development-governor` commands.
- A zero-model, temporary-repository demo.
- External enrollment, task capsules, bounded leases, and terminal receipts.
- Exact allowed/protected paths and hash-bound acceptance material.
- Deterministic verification and Owner-distinct abort semantics.
- A supported Codex default-entry Hook with reversible activation.
- Contract-gated native multi-agent lanes with independent acceptance.
- CI across Python 3.9, 3.11, and 3.12.

## Beta boundary

This is not an OS sandbox, an official OpenAI product, or evidence of measured token,
cost, quality, or productivity improvements. Hook coverage and native worker controls
depend on current Codex surfaces. Use the demo first and adopt it on a disposable or
low-risk project before broader activation.

## Verification

The release is published only after the repository unit suite, the separately frozen
release-core acceptance, a clean wheel install, CLI smoke tests, and the zero-model
demo pass from the tagged commit.
