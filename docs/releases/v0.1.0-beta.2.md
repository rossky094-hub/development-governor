# Development Governor v0.1.0-beta.2

This targeted beta repairs two verification-boundary defects found immediately after
the first public prerelease.

## Changes

- `governor verify` now returns a non-zero process exit status whenever the frozen
  result is `verification_failed`; the JSON receipt remains the authoritative detail.
- The timeout/no-retry regression now proves the invariant from deterministic
  controller receipt fields rather than requiring a child process to write a marker
  before a one-second deadline.

## Non-changes

- Runtime timeout semantics, retry policy, agent topology, and acceptance authority
  are unchanged.
- The v0.1.0-beta.1 tag and release remain immutable.
- This beta still makes no measured token, cost, quality, or productivity claim.
